# Contributing to The Serpent Kingdom

First off, thank you for considering contributing to The Serpent Kingdom! It's people like you that make open source such a great community to learn, inspire, and create.

We want to make contributing to this project as easy and transparent as possible, whether it's:
- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## We Develop with Github

We use GitHub to host code, to track issues and feature requests, as well as accept pull requests.

## Our Labeling System

We use a comprehensive set of **27 labels across 4 groups** to help you find the perfect issue to work on:

1. **Difficulty**: `good first issue`, `beginner-friendly`, `intermediate`, `advanced`
2. **Type**: `bug`, `enhancement`, `documentation`, `performance`, `refactor`, `UI/UX`, `testing`, `security`, `accessibility`
3. **Domain**: `gameplay`, `audio`, `visual-effects`, `boss-fight`, `enemy-AI`, `progression`, `mobile`, `biome`, `save-system`, `multiplayer`
4. **Status**: `help wanted`, `up-for-grabs`, `hacktoberfest`

## Pull Request Process

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes (if available).
5. Make sure your code lints.
6. Issue that pull request!

## Any questions?

If you have any questions, please feel free to open a discussion or ask in the related issue thread. We are happy to help you get started!
